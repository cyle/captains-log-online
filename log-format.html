Activities List Format

For every discreet activity you perform, start a new line!

For each of those activities, each on a separate single line, you can include the following:

  • If you put a dash (-) and then a space at the start of the line, the parser will ignore it. It just makes the list easier for you to read, so feel free to use it.
  • The time spent doing the thing, in brackets, for time tracking. Examples: [1 hour] or [1h30m] or [9:00am - 1:00pm] or [9am to 11am]. All of those are valid and will be parsed.
  • A Footprints Help Desk ticket ID or RFC ID (from the Change Management workspace) associated with it, enclosed in curly braces. Regular old Help Desk tickets can look like this: {#27654} or {27654}, but RFCs you need to prefix with "RFC" like so: {RFC#282} or {RFC252}.
  • If you include the word "meeting" in the line item, the time will also count as a meeting!

An example:

- worked on a ticket [1 hour] {27762}
- change management meeting, lots of fun [20 mins]
- did something that required no time!
- finished up a project {RFC#200}
- weekly meeting lasted forever [2h15m]

Notes Format

Your notes can be in any format you want; they're not parsed, only stored for later reference.
